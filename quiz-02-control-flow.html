ง่าย
ข้อ 1
คำสั่ง if ใช้ทำอะไร?
📝 คำอธิบาย:

if เป็นคำสั่งพื้นฐานในการควบคุมการทำงาน (Control Flow) เพื่อให้โปรแกรมเลือกทำหรือไม่ทำชุดคำสั่งตาม "เงื่อนไข" ที่กำหนด

ง่าย
ข้อ 2
สัญลักษณ์ สี่เหลี่ยมข้าวหลามตัด (Diamond) ใน Flowchart ใช้แทนอะไร?
📝 คำอธิบาย:

ใน Flowchart สี่เหลี่ยมข้าวหลามตัด (Diamond) จะมีทางออก 2 ทางเสมือน (เช่น ใช่/ไม่ใช่ หรือ จริง/เท็จ) ใช้สำหรับการตัดสินใจตรวจสอบเงื่อนไข

ง่าย
ข้อ 3
สัญลักษณ์ วงรี (Oval/Terminal) ใช้แทนอะไร?
📝 คำอธิบาย:

Terminal (รูปวงรี) ใช้ระบุจุดเริ่มต้น (Start) และจุดสิ้นสุด (End/Stop) ของผังงาน

ง่าย
ข้อ 4
สัญลักษณ์ สี่เหลี่ยมผืนผ้า (Rectangle) ใช้แทนอะไร?
📝 คำอธิบาย:

สี่เหลี่ยมผืนผ้า ใช้แทนการปฏิบัติงาน การคำนวณ หรือการกำหนดค่าตัวแปร (Process)

ง่าย
ข้อ 5
สัญลักษณ์ สี่เหลี่ยมด้านขนาน (Parallelogram) ใช้แทนอะไร?
📝 คำอธิบาย:

สี่เหลี่ยมด้านขนาน ใช้เมื่อมีการนำเข้าข้อมูล (Input) หรือแสดงผลข้อมูล (Output) โดยไม่ระบุอุปกรณ์เฉพาะเจาะจง

ง่าย
ข้อ 6
ลูกศร (Arrow) ใน Flowchart บอกอะไรเรา?
📝 คำอธิบาย:

เส้นลูกศร (Flow Line) ใช้เชื่อมต่อสัญลักษณ์ต่างๆ เพื่อแสดงลำดับทิศทางการทำงานจากขั้นตอนหนึ่งไปยังอีกขั้นตอนหนึ่ง

ง่าย
ข้อ 7
คำสั่ง switch ใช้เปรียบเทียบกับอะไร?
📝 คำอธิบาย:

switch เหมาะสำหรับการตรวจสอบค่าของตัวแปรเทียบกับ ค่าคงที่ (Constant values) หลายๆ กรณี (Case) ที่เป็นไปได้

ง่าย
ข้อ 8
คำสั่ง break ใน switch ทำหน้าที่อะไร?
📝 คำอธิบาย:

break จะสั่งให้โปรแกรม หยุด การทำงานใน switch case นั้นและกระโดดออกจาก block ของ switch ทันที ถ้าไม่ใส่ โปรแกรมจะไหลไปทำ case ถัดไปต่อ (Fall-through)

ง่าย
ข้อ 9
สัญลักษณ์ != หมายถึง?
📝 คำอธิบาย:

!= คือเครื่องหมายเปรียบเทียบว่า ไม่เท่ากัน (Not Equal)

ง่าย
ข้อ 10
default ใน switch ทำงานเมื่อใด?
📝 คำอธิบาย:

default จะทำงานเมื่อค่าของตัวแปร ไม่ตรงกับ case ใดเลย ที่กำหนดไว้ (เหมือนกับ else สุดท้ายใน if-else)

ปานกลาง
ข้อ 11
ผลลัพธ์ของ true && false คือ?
📝 คำอธิบาย:

&& คือ AND (และ) ผลลัพธ์จะเป็นจริงก็ต่อเมื่อ ทั้งสองฝั่งเป็นจริง กรณีนี้มี false 1 ตัว ผลจึงเป็น false

ปานกลาง
ข้อ 12
ผลลัพธ์ของ true || false คือ?
📝 คำอธิบาย:

|| คือ OR (หรือ) ผลลัพธ์จะเป็นจริงถ้า มีฝั่งใดฝั่งหนึ่งเป็นจริง กรณีนี้มี true 1 ตัว ผลจึงเป็น true

ปานกลาง
ข้อ 13
ผลลัพธ์ของ !true คือ?
📝 คำอธิบาย:

! คือ NOT (นิเสธ) ใช้กลับค่าความจริง จาก true เป็น false และจาก false เป็น true

ปานกลาง
ข้อ 14
ถ้า x = 5 ผลลัพธ์ของ x > 3 && x < 10 คือ?
📝 คำอธิบาย:

5 มากกว่า 3 (จริง) และ 5 น้อยกว่า 10 (จริง) -> จริง และ จริง = จริง (true)

ปานกลาง
ข้อ 15
Ternary Operator มีรูปแบบอย่างไร?
📝 คำอธิบาย:

Ternary Operator เป็นการเขียน if-else แบบย่อ รูปแบบคือ (condition) ? true_val : false_val

ปานกลาง
ข้อ 16
ผลลัพธ์ของ (5 > 3) ? "A" : "B" คือ?
📝 คำอธิบาย:

5 มากกว่า 3 เป็น จริง ดังนั้นจะเลือกค่าหน้าโคลอน (:) ซึ่งคือ "A"

ปานกลาง
ข้อ 17
ถ้าลืมใส่ break ใน switch จะเกิดอะไรขึ้น?
📝 คำอธิบาย:

เกิดอาการที่เรียกว่า Fall-through คือโปรแกรมจะไหลไปทำงานใน case ถัดไปเรื่อยๆ (แม้เงื่อนไขจะไม่ตรง) จนกว่าจะเจอ break หรือจบ switch

ปานกลาง
ข้อ 18
ข้อใดถูกต้องเกี่ยวกับ else if?
📝 คำอธิบาย:

else if ใช้เพื่อตรวจสอบเงื่อนไขเพิ่มเติมเมื่อ if แรกเป็นเท็จ เราสามารถมี else if กี่อันก็ได้ตามต้องการ

ปานกลาง
ข้อ 19
สัญลักษณ์ >= หมายถึง?
📝 คำอธิบาย:

>= คือ Greater than or Equal to (มากกว่า หรือ เท่ากับ)

ปานกลาง
ข้อ 20
ผลลัพธ์ของ false || false คือ?
📝 คำอธิบาย:

OR (||) จะเป็นจริงถ้ามีอย่างน้อย 1 ตัวจริง แต่ถ้า เท็จทั้งคู่ ผลลัพธ์จะเป็น false

ยาก
ข้อ 21
ผลลัพธ์ของโค้ดนี้? 
int x = 10; if (x > 5) System.out.print("A"); System.out.print("B");
📝 คำอธิบาย:

เงื่อนไข x > 5 เป็นจริง พิมพ์ "A" ออกมา ส่วนคำสั่งพิมพ์ "B" ไม่อยู่ใน if (เพราะไม่มีปีกกา และอยู่บรรทัดถัดไปโดยไม่มี Indent ใน Java ก็นับเป็นนอก if) ดังนั้นจึงทำ "B" เสมอ รวมได้ AB

ยาก
ข้อ 22
ผลลัพธ์ของโค้ดนี้? 
int x = 3; if (x > 5) System.out.print("A"); System.out.print("B");
📝 คำอธิบาย:

เงื่อนไข x > 5 เป็น เท็จ (3 ไม่มากกว่า 5) จึงข้ามบรรทัดพิมพ์ "A" ไป ส่วนบรรทัดพิมพ์ "B" อยู่นอก if จึงทำงานตามปกติ ได้ผลลัพธ์ B

ยาก
ข้อ 23
ผลลัพธ์ของ !(true && false) คือ?
📝 คำอธิบาย:

ทำในวงเล็บก่อน: true && false ได้ false จากนั้นเจอ ! (NOT) ข้างหน้า กลับค่า false ให้กลายเป็น true

ยาก
ข้อ 24
ข้อใดใช้เป็น expression ใน switch ได้ใน Java?
📝 คำอธิบาย:

Java รุ่นใหม่ๆ รองรับการใช้ String ใน switch case ได้ แต่ยัง ไม่รองรับ double, float หรือ boolean

ยาก
ข้อ 25
ผลลัพธ์ของโค้ด? 
int x = 5; if (x = 10) { ... }
📝 คำอธิบาย:

ใน Java if ต้องการค่า boolean แต่ x = 10 เป็นการ กำหนดค่า (expression returns int) ไม่ใช่การเปรียบเทียบ (==) จึงเกิด Compile Error (Incompatible types)

ยาก
ข้อ 26
Short-circuit evaluation หมายถึง?
📝 คำอธิบาย:

Short-circuit (&&, ||) จะหยุดการทำงานทันทีที่รู้ผลลัพธ์แน่นอนแล้ว เช่น false && ... จะไม่ทำด้านหลังต่อเพราะรู้ว่าเป็นเท็จแน่ๆ

ยาก
ข้อ 27
ถ้า false && method() method() จะถูกเรียกหรือไม่?
📝 คำอธิบาย:

ด้วยกฎ Short-circuit เมื่อเจอ false && ผลลัพธ์รวมจะเป็น false แน่นอน Java จึงไม่เสียเวลารัน method() ด้านหลัง

ยาก
ข้อ 28
ถ้า true || method() method() จะถูกเรียกหรือไม่?
📝 คำอธิบาย:

เมื่อเจอ true || ผลลัพธ์รวมจะเป็น true แน่นอน (Short-circuit) Java จึงไม่รัน method() ด้านหลัง

ยาก
ข้อ 29
ผลลัพธ์ของโค้ด? 
int x = 1; switch(x) { case 1: print("A"); case 2: print("B"); }
📝 คำอธิบาย:

เข้า case 1 พิมพ์ "A" แต่ไม่มี break; โปรแกรมจึงไหลลงไปทำ case 2 ต่อและพิมพ์ "B" ด้วย (Fall-through)

ยาก
ข้อ 30
ผลลัพธ์ของ (10 > 5) ? (3 > 2) ? "A" : "B" : "C" คือ?
📝 คำอธิบาย:

Nested Ternary: 10>5 จริง -> เข้าไปทำตัวหน้า (3 > 2) ? "A" : "B"; 3>2 จริง -> ตอบ "A"
