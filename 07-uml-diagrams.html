UML Class & Object Diagrams

📅 4 กุมภาพันธ์ 2026 ⏱️ อ่าน 25 นาที

UML (Unified Modeling Language) คือภาษามาตรฐานในการเขียนแผนภาพ เพื่อออกแบบและสื่อสารโครงสร้างของโปรแกรมก่อนลงมือเขียนโค้ดจริง

UML คืออะไร?

UML = Unified Modeling Language
เป็นภาษาสำหรับวาดแผนภาพที่ใช้กันทั่วโลก ช่วยให้โปรแกรมเมอร์เข้าใจโครงสร้างของระบบได้ง่ายขึ้น

ประเภทของ UML Diagram

1. Class Diagram

แสดงโครงสร้าง Class และความสัมพันธ์

2. Object Diagram

แสดง Object จริงที่สร้างจาก Class

3. Sequence Diagram

แสดงลำดับการทำงาน

4. Use Case Diagram

แสดงการใช้งานระบบ

Class Diagram

แสดงโครงสร้างของ Class ประกอบด้วย 3 ส่วน:

┌─────────────────────────┐
│      ClassName          │  ← ชื่อ Class
├─────────────────────────┤
│ - name: String          │  ← Attributes
│ - age: int              │
│ # balance: double       │
├─────────────────────────┤
│ + getName(): String     │  ← Methods
│ + setName(n: String)    │
│ - calculate(): int      │
└─────────────────────────┘

สัญลักษณ์ Access Modifier

+ public     // เข้าถึงได้จากทุกที่
- private    // เข้าถึงได้เฉพาะใน Class
# protected  // เข้าถึงได้ใน Class และ Subclass
~ package    // เข้าถึงได้ใน Package เดียวกัน

ตัวอย่าง Class Diagram

┌─────────────────────────┐
│        Student          │
├─────────────────────────┤
│ - studentId: String     │
│ - name: String          │
│ - gpa: double           │
├─────────────────────────┤
│ + Student(id, name)     │
│ + getName(): String     │
│ + setName(n: String)    │
│ + calculateGrade(): char│
└─────────────────────────┘

แปลงเป็นโค้ด Java:

public class Student {
    private String studentId;
    private String name;
    private double gpa;
    
    public Student(String id, String name) {
        this.studentId = id;
        this.name = name;
    }
    
    public String getName() {
        return name;
    }
    
    public void setName(String n) {
        this.name = n;
    }
    
    public char calculateGrade() {
        if (gpa >= 3.5) return 'A';
        else if (gpa >= 2.5) return 'B';
        else return 'C';
    }
}

Object Diagram

แสดง Instance จริง ที่สร้างจาก Class พร้อมค่าข้อมูล:

┌─────────────────────────────┐
│   somchai: Student        │  ← ชื่อ object : ชื่อ Class
├─────────────────────────────┤
│ studentId = "64010001"      │  ← ค่าจริงใน attribute
│ name = "สมชาย"              │
│ gpa = 3.75                  │
└─────────────────────────────┘
⚠️ ความแตกต่าง Class vs Object Diagram
  • Class Diagram: แสดงโครงสร้าง (ชนิดข้อมูล เช่น String, int)
  • Object Diagram: แสดงค่าจริง (ค่าข้อมูล เช่น "สมชาย", 3.75)

Relationship ใน Class Diagram

1. Association (ความสัมพันธ์ทั่วไป)

แสดงว่า Class หนึ่งรู้จักหรือใช้งาน Class อื่น:

┌──────────┐         ┌──────────┐
│ Student  │─────────│  Course  │
└──────────┘         └──────────┘
   นักเรียน ลงทะเบียน วิชาเรียน

2. Inheritance / Generalization (การสืบทอด)

ใช้ลูกศรหัวสามเหลี่ยมกลวง ชี้ไปหา Parent:

       ┌──────────┐
       │  Animal  │  ← Parent Class
       └────△─────┘
            │
     ┌──────┴──────┐
     │             │
┌────┴───┐   ┌─────┴────┐
│   Dog  │   │   Cat    │  ← Child Classes
└────────┘   └──────────┘

3. Aggregation (มี/ประกอบด้วย - แบบหลวม)

ใช้เพชรกลวง แสดงว่า "มี" แต่แยกจากกันได้:

┌───────────┐       ┌──────────┐
│Department │◇──────│ Employee │
└───────────┘       └──────────┘
  แผนก "มี" พนักงาน (แต่ลบแผนก พนักงานยังอยู่)

4. Composition (มี/ประกอบด้วย - แบบแน่น)

ใช้เพชรทึบ แสดงว่า "มี" และแยกจากกันไม่ได้:

┌──────────┐       ┌──────────┐
│  House   │◆──────│   Room   │
└──────────┘       └──────────┘
  บ้าน "มี" ห้อง (ลบบ้าน ห้องก็หายไปด้วย)

5. Dependency (พึ่งพา)

ใช้ลูกศรประ แสดงว่า Class หนึ่งใช้ Class อื่นชั่วคราว:

┌──────────┐       ┌──────────┐
│  Order   │- - - >│ Product  │
└──────────┘       └──────────┘
  Order ใช้ Product (เป็น parameter)

Multiplicity (จำนวน)

1      // มี 1 ตัว
0..1   // มี 0 หรือ 1 ตัว
*      // มีกี่ตัวก็ได้ (0 หรือมากกว่า)
1..*   // มีอย่างน้อย 1 ตัว
0..*   // มี 0 หรือมากกว่า (เหมือน *)
n..m   // มี n ถึง m ตัว เช่น 2..5
┌──────────┐ 1    * ┌──────────┐
│ Teacher  │────────│ Student  │
└──────────┘        └──────────┘
  ครู 1 คน สอน นักเรียน หลายคน (*)

สรุปสัญลักษณ์ที่ต้องจำ

📝 สรุป UML สำหรับสอบ

  • + public, - private, # protected, ~ package
  • Class Diagram: แสดงโครงสร้าง Class (ชื่อ, Attributes, Methods)
  • Object Diagram: แสดง Instance จริง พร้อมค่าข้อมูล
  • △ Inheritance: ลูกศรหัวสามเหลี่ยมกลวง ชี้ไปหา Parent
  • ◇ Aggregation: เพชรกลวง (has-a แบบหลวม)
  • ◆ Composition: เพชรทึบ (has-a แบบแน่น)
  • Multiplicity: 1, 0..1, *, 1..*, n..m
