OOP: Object-Oriented Programming

📅 2 กุมภาพันธ์ 2026 ⏱️ อ่าน 20 นาที

OOP (Object-Oriented Programming) คือแนวคิดในการเขียนโปรแกรมโดยมอง ทุกอย่างเป็นวัตถุ (Object) ที่มีคุณสมบัติและความสามารถ

4 หลักการสำคัญของ OOP

1. Encapsulation

ซ่อนข้อมูลภายใน ควบคุมการเข้าถึง

2. Inheritance

สืบทอดคุณสมบัติจาก class อื่น

3. Polymorphism

method เดียวกัน ทำงานต่างกันได้

4. Abstraction

ซ่อนความซับซ้อน แสดงเฉพาะที่จำเป็น

Class และ Object

Class = พิมพ์เขียว/แม่พิมพ์ (เช่น แบบบ้าน)
Object = ของจริงที่สร้างจาก class (เช่น บ้านจริงๆ)

สร้าง Class

public class Student {
    // Attributes (คุณสมบัติ)
    String name;
    int age;
    double gpa;
    
    // Method (ความสามารถ)
    void study() {
        System.out.println(name + " กำลังอ่านหนังสือ");
    }
    
    void introduce() {
        System.out.println("สวัสดี ผมชื่อ " + name + " อายุ " + age);
    }
}

สร้าง Object

public class Main {
    public static void main(String[] args) {
        // สร้าง object จาก class Student
        Student s1 = new Student();
        s1.name = "สมชาย";
        s1.age = 20;
        s1.gpa = 3.5;
        
        Student s2 = new Student();
        s2.name = "สมหญิง";
        s2.age = 19;
        
        // เรียกใช้ method
        s1.introduce(); // สวัสดี ผมชื่อ สมชาย อายุ 20
        s2.study();      // สมหญิง กำลังอ่านหนังสือ
    }
}

Constructor

Method พิเศษที่ทำงานตอนสร้าง object ใหม่:

public class Student {
    String name;
    int age;
    
    // Constructor - ชื่อต้องเหมือน class
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

// สร้าง object พร้อมกำหนดค่า
Student s1 = new Student("สมชาย", 20);
⚠️ this คืออะไร?
this หมายถึง object ปัจจุบัน ใช้แยกระหว่าง attribute กับ parameter ที่ชื่อเหมือนกัน

Encapsulation (การห่อหุ้มข้อมูล)

ซ่อน attribute ด้วย private แล้วเข้าถึงผ่าน getter/setter:

public class BankAccount {
    private double balance; // ซ่อนไว้!
    
    // Getter - อ่านค่า
    public double getBalance() {
        return balance;
    }
    
    // Setter - กำหนดค่า (มีเงื่อนไข)
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }
}

Inheritance (การสืบทอด)

สร้าง class ใหม่โดยสืบทอดจาก class เดิม ใช้ extends:

// Parent class
public class Animal {
    String name;
    
    void eat() {
        System.out.println(name + " กำลังกิน");
    }
}

// Child class
public class Dog extends Animal {
    void bark() {
        System.out.println(name + ": โฮ่ง โฮ่ง!");
    }
}

// ใช้งาน
Dog d = new Dog();
d.name = "บ๊อบ";
d.eat();  // บ๊อบ กำลังกิน (สืบทอดมา)
d.bark(); // บ๊อบ: โฮ่ง โฮ่ง! (ของตัวเอง)

Polymorphism (หลายรูปแบบ)

Method เดียวกันแต่ทำงานต่างกันใน class ลูก (Override):

public class Animal {
    void makeSound() {
        System.out.println("...");
    }
}

public class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("โฮ่ง โฮ่ง!");
    }
}

public class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("เมี๊ยว~");
    }
}

// Polymorphism!
Animal a1 = new Dog();
Animal a2 = new Cat();
a1.makeSound(); // โฮ่ง โฮ่ง!
a2.makeSound(); // เมี๊ยว~
💡 @Override ทำไมต้องใส่?
ช่วยให้ compiler ตรวจสอบว่าเรา override method จริงๆ ถ้าพิมพ์ชื่อผิดจะ error

Access Modifiers

public    // เข้าถึงได้จากทุกที่
private   // เข้าถึงได้เฉพาะใน class นี้
protected // เข้าถึงได้ใน class นี้ + class ลูก
(default)  // เข้าถึงได้ใน package เดียวกัน

📝 สรุป OOP สำหรับสอบ

  • Class = แม่พิมพ์, Object = ของจริง
  • Constructor = สร้าง object พร้อมค่าเริ่มต้น
  • Encapsulation = private + getter/setter
  • Inheritance = extends สืบทอด
  • Polymorphism = @Override ทำต่างกัน
  • this = object ปัจจุบัน
